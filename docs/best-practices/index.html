最佳实践

团队使用 Claude Code 的最佳实践

工作流程

1. 始终从根目录运行

在项目根目录启动 Claude Code，确保完整的项目上下文。

cd my-project
claude

2. 新项目先运行 /init

让 Claude Code 扫描项目并创建初始配置。

3. 复杂任务使用 Plan Mode

新功能、架构变更等复杂任务，先进入 Plan Mode 规划。

4. Git 是你的安全网

重要操作前先提交，出问题可以随时回退。

CLAUDE.md 配置

保持简洁

只包含项目特定的关键信息，避免指令过载。

使用 Critical Rules

将不可违反的规则放在 Critical Rules 部分，最高优先级。

提供验证方式

告诉 Claude 如何验证它的工作（测试命令、构建命令等）。

模型选择

模型 适用场景
Haiku 简单查询、轻量任务、快速响应
Sonnet 日常开发（默认推荐）
Opus 复杂架构决策、深度分析

团队协作

提交 CLAUDE.md

将 CLAUDE.md 纳入版本控制，团队共享配置。

代码审查包含 AI 生成内容

Claude 生成的代码也需要经过正常审查流程。

记录 Claude 协助的内容

提交信息中标注由 Claude 生成或协助的代码。
