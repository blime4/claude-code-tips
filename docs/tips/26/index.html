Tip #26 intermediate Act 2

Start features in Plan Mode

Always start new features in Plan Mode

⏱ 18:27 📁 workflow
标签： planningworkflowfeatures

描述

开始新功能开发时，先进入 Plan Mode 进行规划。

为什么重要

Plan Mode 让你：

  • 先思考再动手
  • 识别潜在问题
  • 设计合理的方案
  • 避免返工

工作流程

1. 接收需求
2. 进入 Plan Mode
3. 探索代码库
4. 设计方案
5. 获取批准
6. 实施方案

触发方式

# 自动触发
> 我需要添加用户认证功能
# Claude 检测到复杂任务，自动进入 Plan Mode

# 手动触发
/plan

适合 Plan Mode 的任务

  • 新功能实现
  • 架构变更
  • 复杂重构
  • 多文件修改
