## Description

Claude can analyze your codebase to provide insights, identify patterns, and suggest improvements.

## Usage

Ask Claude to analyze your project:

```bash
# Analyze patterns in your codebase
"Identify common patterns in my codebase"

# Extract architecture insights
"What architectural patterns are used here?"

# Find potential issues
"Identify potential security issues"

# Suggest improvements
"What improvements would you recommend?"
```

## What Claude Can Find

- **Code patterns**: Repeated structures, anti-patterns, idioms
- **Architecture**: Design patterns, coupling, cohesion
- **Issues**: Bugs, security vulnerabilities, performance problems
- **Improvements**: Refactoring opportunities, best practices

## Best Practices

1. **Be specific**: Focus on particular areas or concerns
2. **Provide context**: Explain what you're looking for
3. **Ask for alternatives**: Request multiple approaches
4. **Validate findings**: Review suggestions before implementing

## Example Queries

```bash
# Code quality
"What code quality issues exist in src/?"

# Performance
"Are there any performance bottlenecks?"

# Security
"Review authentication flows for security issues"

# Architecture
"How is state managed across components?"
```
